Documentation

Minimum Requirements

  • A minimum of 1 computer with Debian or Ubuntu installed to act a the frontend of the Cluster. This frontend, needs at least 1 Net interface. The tipical case is to have 2 NET interfaces. 1 NET interface for the internal NET and the other interface to have an exit to the world, Internet.

    This frontend will have a DHCP server for the internat NET so an extra care is needed in order not to have problems between the different DHCP servers

  • The computers that are going to be slave nodes, don't have to be homogeneus (same hardware) don't need even a HD. They need a NET interface and have the BIOS configured to boot from it.

    • Installing

  • Add repository
    • sudo add-apt-repository ppa:kestrel/kestrelhpc
  • Install KestrelHPC
    • sudo apt-get install kestrelhpc kestrelhpc-openmpi

    Adding slave nodes

  • First we create an image for the slave nodes
    • kestrel-images --new image1
  • We register the desired nodes
    • kestrel-control --register
  • We turn on the slave nodes that we want
  • We have finished registering the nodes
    • kestrel-control --list-registered

    Configured Libraries

    In this first version openMPI is configured for its use, but in the future version, the intention is to have the Charm++ configuration automatically done also. Moreover, thanks to the modulized design of KestrelHPC, third-party packages could be created to have other library and languages configured in the system.

    An example of the openMPI command.

    kestrel-launchmpi mpi_program

    Examples commands

  • Here there some of the most used commands
  • Nodes booting/controlling
    • kestrel-control --wake "image1" # Turn on the nodes of image1
    kestrel-control --list-connected "image1" # Check if the nodes where correctly booted
    kestrel-control --halt "image1" # Halt the nodes that are part of the group image1
  • Modifying images
    • kestrel-apt --install "vim" --remove "nano" # Installs vim and removes nano from the default image
    kestrel-images --chroot "image1" # Do chroot in the image1. 
				# By default chroot is disable due to security policies
